## 2. v0.3.0 - 架构重构版本

### 2.1 版本目标

v0.3.0 是一个以技术债务偿还和架构升级为主要目标的版本，通过引入 Clean Architecture 提升代码质量和可维护性。

**主要目标**:
1. 重构代码架构，采用 Clean Architecture 分层设计
2. 提升测试覆盖率，建立完善的测试体系
3. 优化性能和内存使用
4. 为后续功能扩展奠定坚实基础

### 2.2 架构重构详情

#### 重构前结构 (v0.2.x)

```
src/
├── main.rs
├── lib.rs
├── app.rs          # 所有逻辑混在一起
├── gallery.rs
├── viewer.rs
├── decoder.rs
└── utils.rs
```

#### 重构后结构 (v0.3.0)

```
src/
├── main.rs              # UI 层 - 入口点
├── config.rs            # Domain 层 - 配置
├── app/                 # Domain 层 - 核心应用
│   └── mod.rs
├── viewer/              # Application 层 - 查看器
│   └── mod.rs
├── gallery/             # Application 层 - 图库
│   └── mod.rs
├── decoder/             # Application 层 - 解码器
│   └── mod.rs
├── dnd/                 # Infrastructure 层 - 拖放
│   └── mod.rs
├── clipboard/           # Infrastructure 层 - 剪贴板
│   └── mod.rs
├── info_panel.rs        # UI 层 - 信息面板
├── shortcuts_help.rs    # UI 层 - 快捷键帮助
└── utils/               # Infrastructure 层 - 工具
    ├── mod.rs
    ├── errors.rs
    └── threading.rs
```

#### 分层说明

| 层级 | 模块 | 设计原则 |
|------|------|----------|
| **Domain** | `app`, `config` | 核心业务逻辑，独立于框架 |
| **Application** | `viewer`, `gallery`, `decoder` | 用例实现，编排业务逻辑 |
| **Infrastructure** | `utils`, `dnd`, `clipboard` | 技术细节，可替换实现 |
| **UI** | `main.rs`, `info_panel`, `shortcuts_help` | 用户界面，最易变化 |

#### F-003: 基础浏览

**功能描述**: 图片基础浏览功能

**详细需求**:
1. **自适应窗口**: 图片自动适应窗口大小显示
2. **1:1 显示**: 
   - 快捷键：`1` 或 `Ctrl+1`
   - **工具栏按钮**: 添加 "1:1" 按钮，点击直接切换到原始尺寸
   - 显示当前缩放比例（如 "100%"）
3. **上下张切换**: ←/→ 箭头键或工具栏按钮
4. **缩放控件**:
   - 工具栏 `+` / `-` 按钮
   - 或下拉菜单选择百分比（10%, 25%, 50%, 75%, 100%, 150%, 200%, 400%, 800%, 1600%）
   - 或可输入的缩放百分比框（支持直接输入 10-2000%）
5. **当前缩放比例实时显示**: 状态栏或工具栏显示当前百分比

**验收标准**:
- ✅ 1:1 按钮可点击切换到原始尺寸
- ✅ 缩放控件可精确控制缩放比例
- ✅ 当前缩放比例实时更新显示

### 6.1 v0.3.1 - v0.4.0 功能规划

#### 6.1.1 1:1 显示按钮
- 工具栏添加 "1:1" 按钮
- 点击直接切换到原始尺寸显示
- 快捷键：`1` 或 `Ctrl+1`

#### 6.1.2 缩放控件增强
- 工具栏添加 `+` / `-` 按钮
- 或下拉菜单选择百分比（10%, 25%, 50%, 75%, 100%, 150%, 200%, 400%, 800%, 1600%）
- 或可输入的缩放百分比框（支持直接输入数值）
- 当前缩放比例实时显示（如："100%"）

#### 6.1.3 多语言支持（i18n）

**架构设计**：
- 使用 Rust i18n 库（`fluent` 或 `i18n-embed`）
- 语言文件格式：`.ftl` (Fluent) 或 `.json`
- 目录结构：`locales/en-US/`, `locales/zh-CN/`

**第一阶段：中英文双语**：
- 所有UI文本提取到语言文件
- 支持语言切换（菜单或配置）
- 默认跟随系统语言，可手动设置

**扩展性保留**：
- 语言文件标准化，便于添加新语言
- 支持运行时语言切换（无需重启）
- 预留其他语言接口
