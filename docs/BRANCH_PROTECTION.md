# 分支保护规则

本文档描述本仓库配置的分支保护规则。

## 受保护的分支

### `main` (生产环境)

主分支包含生产就绪的代码。

**保护规则：**

| 规则 | 设置 | 描述 |
|------|------|------|
| 要求拉取请求审查 | ✅ 1 个批准 | 所有变更必须经过审查 |
| 取消过时审查 | ✅ 已启用 | 新提交后取消审查 |
| 要求状态检查 | ✅ 已启用 | 合并前 CI 必须通过 |
| 要求分支保持最新 | ✅ 已启用 | 合并前必须变基 |
| 必需状态检查 | `fmt`, `clippy`, `test`, `coverage` | 所有 CI 检查必须通过 |
| 包含管理员 | ✅ 已启用 | 规则适用于管理员 |

**允许的操作：**
- ✅ 合并拉取请求（通过合并按钮）
- ✅ 强制推送（仅限管理员紧急情况）
- ❌ 直接推送（必须通过 PR）
- ❌ 删除分支

### `develop` (开发集成)

开发分支用于集成新功能。

**保护规则：**

| 规则 | 设置 | 描述 |
|------|------|------|
| 要求拉取请求审查 | ✅ 1 个批准 | 所有变更必须经过审查 |
| 取消过时审查 | ✅ 已启用 | 新提交后取消审查 |
| 要求状态检查 | ✅ 已启用 | CI 必须通过 |
| 必需状态检查 | `fmt`, `clippy`, `test` | 核心检查必须通过 |

## 分支命名规范

### 功能分支
```
feature/<功能描述>
```

### 修复分支
```
fix/<问题描述>
```

### 发布分支
```
release/<版本号>
```

### 热修复分支
```
hotfix/<问题描述>
```

## 状态检查说明

- `fmt` - 代码格式化
- `clippy` - 静态分析
- `test` - 单元测试
- `coverage` - 代码覆盖率
